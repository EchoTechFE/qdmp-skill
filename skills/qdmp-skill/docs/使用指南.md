# 千岛小程序 Skill 使用指南

## 概述

千岛小程序 Skill 是一个帮助开发者快速创建、开发和部署千岛小程序的 Claude Code 技能。它提供了完整的开发流程支持，从环境配置到最终部署发布。

## 前置条件

1. 注册 [千岛开放平台](https://open.qiandao.com) 账号，创建小程序获取 `appId` 和 `appSecret`
2. 安装 [Node.js](https://nodejs.org)（建议 v18+）
3. 安装 Claude Code

---

## 一、环境配置

### 1. 安装开发工具

```bash
# 安装 pnpm
npm install -g pnpm

# 安装 qdmp-cli
npm install -g qdmp-cli

# 验证安装
qdmp-cli --version
pnpm --version
```

### 2. 添加 MCP 服务

```bash
claude mcp add --transport http qdmp-gitlab https://openapi.qiandao.com/gitlab/mcp
claude mcp add --transport http qdmp-aliyun https://openapi.qiandao.com/aliyun/mcp
```

### 3. 安装 Skill

```bash
# 添加插件市场
/plugin marketplace add https://github.com/EchoTechFE/qdmp-skill.git

# 安装插件
/plugin install qdmp-skill
```

## 二、快速开始

启动 Claude Code 后，使用 `/qdmp-skill` 触发千岛小程序开发助手。

### 创建项目

```
创建一个千岛小程序项目，项目名叫 myShop
```

Claude 会自动：查看可用模板 → 创建项目目录 → 安装依赖 → 生成前后端结构 → 创建 `qdmp.json` 配置文件。

项目结构：

```
myShop/
├── frontend/
│   ├── src/
│   │   ├── pages/        # 页面
│   │   ├── components/   # 组件
│   │   ├── composables/  # 组合式函数
│   │   └── utils/        # 工具函数
│   ├── package.json
│   └── qdmp.json         # 小程序配置（appId、appSecret、runtime）
└── backend/              # 后端服务（Go / Node.js / Python，监听 :8080）
```

### 开发调试

**全量调试（推荐）**：

```
部署测试环境
```

Claude 会先启动本地后端服务，健康检查通过后自动将前端 `BACKEND_BASE` 替换为 `/api`，再启动前端 dev server。

```
本地测试环境已就绪

  前端 dev server: http://localhost:10086
  后端 API:        http://localhost:8080
```

**单独调试**：

```
部署后端测试环境    # 仅启动后端
部署前端测试环境    # 仅启动前端 dev server
```

**真机调试**：

```bash
pnpm run build
qdmp-cli upload
# 使用千岛 App 扫码预览
```

### 发布上线

```
部署上线
```

Claude 会先发布后端（触发 CI/CD Pipeline，自动跟踪构建进度），后端成功后自动将前端 `BACKEND_BASE` 替换为线上后端地址（`https://{appId}.qiandaomp.com`），再打包上传前端。

上传后前往 https://open.qiandao.com 提交审核。

## 四、常见问题

### qdmp-cli 命令未找到

```bash
npm install -g qdmp-cli
```

如果仍然找不到，检查 npm 全局路径是否在 PATH 中：
```bash
npm config get prefix
# 确保 {prefix}/bin 在 PATH 中
```

### 依赖安装失败

确认网络正常，可尝试：
```bash
pnpm install --registry https://registry.npmmirror.com
```

### 登录失败

1. 检查网络连接
2. 确认账号密码正确
3. 重新执行 `qdmp-cli login`
